Flipping Flashcards

javascript · html · css

Inspired by an article I read, which recommended flashcards as a great way to learn annything/as a brilliant tool to learn/for learning anything under the sun singing the praises of flashcards as the best solution for learning anything. This being the 21st century, I was not about to make paper flashcards, but I did have a vision of a card flipping.

01.
The interface Arguably the most fun part. I decided to go a bit crazy with retro cyberpunk style and loved it. I chose jQuery flip as a mechanism for the most importannt part.

02.
Source for questions and answers The first phase was researching different methods of extracting data from spreadsheets into Javascript. It seemed nnatural to use either local Excel files or google Sheets for this purpose. I tesed both and in the process learnt to update Google Sheets remotely through node, but I wasn't happy whith how difficult to parse the result was. What is more, I realized that spreadsheets are neither a natural/convenient not easiest to use solution. In the end, I decided to hard-code the content into index.js for now. I wanted to use a human-readable format for any new card sets, but one that is simple to turn into JavaScript objects for use with the code. · action screenshot · · screenshot of code·

03.
Viewing/Showing/displaying questions and answers

04.
Individual flashcard ratings A crucial part of the project was allowinng for a way for the user to indicate if they want to see the same card again. I decided on rating buttonns Each card is assigned a rating of 6 when created, and a series of array actions update it when a hgbjk function is triggered at the press of a rating button It wouldn't be hard to change the behaviour of the buttons through minor changes in the code. The V button could take a card out of circulationn completely, or the total nnumber of rating points assigned to each card could be lowered to shorten the learning session, or increased so that even cards rated V by the user will be shown 4 or more times

05.
Responsive design This was one of the projects that looked ok when first viewed on mobile, but I decided to abandon the classic landscape rectangular shape on smaller screens. The new flashcard shape is 80% of the viewport width, but no more than 1:1 aspect ratio

06.
what I learned I am still working on this project. I'd really like o find asolution for importing outside data to use with this interface. Because I changed the direction of the project more than once, I learned to appreciate git for making it possible to work on different branches without feeling like i needed to give up one to try another. One day I might even start using flashcards to study./ /Most importantly, I learnt that I donn't really like studying with flashcards/studying with flashcards is nnnot my thinng

Workday Clock

javascript · html · css

I've never liked using round face analog clocks. The two hands (why should the long hand indicate the short time period and not the other way around?), the reversing direction (left to right when on top, but right to left on the bottom), my general desire to be unique and insufferable/an insufferable contrarian... However, watching X's video ... changed my mind to an extent and crystallized an idea: yes to a visual representation of the length of each hour and the whole day, yes toa way to easily visualize time remaining to an appointment, but only one direction of travel, no time change/turns off during off duty hours

Keyboard trapping for enhanced accessibility

Keyboard trapping is normally evil. But, you can use this "anti-pattern" in a good way to enhance the accessibility your application for your users. Just wield the power responsibly!

Clicking the button below will launch a modal window, and apply focus to the first focussable element inside it. JavaScript will not allow the focus to move outside the modal while the modal is active. You can think of the modal as a mini page-within-a-page.
